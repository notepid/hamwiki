# Contributing to the HAM Radio Wiki

This wiki is an open-source project and contributions from the amateur radio community are welcome and encouraged. Whether you want to fix a typo, expand an existing article, or write an entirely new page, your knowledge helps make this resource better for everyone.

## Where the source lives

All wiki content is hosted on GitHub:

**[github.com/notepid/hamwiki](https://github.com/notepid/hamwiki)**

The repository contains the full wiki as markdown files organised into folders that mirror the site structure. Each page uses Wiki.js frontmatter at the top of the file for metadata such as the title, description, and tags.

## How to contribute

### Option 1 — Pull request (preferred)

If you are comfortable with Git and GitHub, the best way to contribute is by submitting a pull request:

1. **Fork** the repository on GitHub.
2. **Clone** your fork to your local machine.
3. **Create a branch** for your changes (e.g. `fix/propagation-typos` or `add/dmr-programming-guide`).
4. **Edit or add** markdown files following the existing structure and frontmatter format.
5. **Commit** your changes with a clear commit message describing what you changed and why.
6. **Push** your branch to your fork and open a **pull request** against the main repository.

A maintainer will review your pull request and may suggest changes before merging. Don't worry about getting everything perfect on the first try — the review process is there to help.

#### Page format

Each page should begin with Wiki.js frontmatter like this:

```yaml
---
title: Your Page Title
description: A short one-line description of the page content
published: true
date: 2026-04-05T12:00:00.000Z
tags: relevant, tags, here
editor: markdown
dateCreated: 2026-04-05T12:00:00.000Z
---
```

The rest of the page is standard markdown. Look at existing pages in the repository for examples of style and structure.

### Option 2 — Email

If you are not familiar with Git or GitHub, you can contribute by sending an email to:

**[wiki@lb1dk.no](mailto:wiki@lb1dk.no)**

Please include the complete content of the new or updated page in your email, preferably formatted in markdown. If you are suggesting changes to an existing page, mention which page you are referring to so the maintainers can find it easily.

## What kinds of contributions are helpful

- **Corrections** — fixing factual errors, outdated information, or broken links
- **Expansions** — adding depth to stub pages or filling in missing details
- **New pages** — writing content on topics not yet covered by the wiki
- **Translations and localisation** — helping ensure information is accurate for operators in different countries
- **Review** — reading through pages flagged as AI-generated and verifying their accuracy

## Guidelines

- Write for a broad audience. Many readers will be new to amateur radio, so explain jargon and link to related pages where helpful.
- Keep an international perspective. Regulations and practices vary by country — note where something is country-specific rather than presenting it as universal.
- Be factual and cite sources where possible, especially for regulatory or technical claims.
- Use plain, clear language. This is a reference wiki, not a textbook — aim for approachable and concise.

## Questions?

If you have questions about contributing or are unsure where a new page should go, feel free to reach out at [wiki@lb1dk.no](mailto:wiki@lb1dk.no) or open an issue on the [GitHub repository](https://github.com/notepid/hamwiki).

Thank you for helping build a better resource for the amateur radio community. 73!
